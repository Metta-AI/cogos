# Writing Style

- **No hyperbole:** Never use "massive", "huge", "extremely", "significant", "major",
  "aggressive", "tremendous". Use specific numbers or factual descriptions.
- **No PR/commit counts:** "17 commits" tells the reader nothing. Describe what was
  built or what changed toward the goal.
- **Summarize, don't list:** Cluster related work into a concise summary of what it
  accomplishes. "Added CUDA advantage support to the training library" not a list of
  3 PRs.
- **Factual assessments:** "No engineering activity this window" not "this thread has
  stalled dramatically."
- **Business-readable:** "Improved training infrastructure reliability" not "fixed CUDA
  OOM from memory fragmentation."
- **Honest:** No activity = say so plainly, don't invent progress.
- **Credit cross-contributions:** Note when others contribute work advancing a thread.
- **Link to evidence:** When claiming work was done, link to the evidence (Asana task,
  repo, demo).
- **No section labels in thread bodies:** Don't use bold headers like "Progress:" or
  "Assessment:". Write two flowing paragraphs — first "Since the last report, ..."
  covering activity, second "Overall, ..." giving a factual assessment.
- **No blank lines between threads:** HEADING_2 style provides visual separation.
